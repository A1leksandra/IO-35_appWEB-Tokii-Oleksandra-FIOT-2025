2. Семантичні компоненти інтерфейсу

2.1. Шапка (header) з логотипом та навігаційним меню

Реалізовано семантичний тег <header> з наступними елементами:

  • Логотип: <div class="logo">Одяг Олександри</div>
  • Навігація: <nav> з <ul class="nav-links">
  • Бургер-меню: <button class="burger-menu"> для мобільних пристроїв
  • Іконки: профіль та кошик покупок

2.2. Основний блок із розташуванням інформаційних карток

Реалізовано через семантичні теги <section>:

  • Hero-секція: головний банер з заголовком та кнопкою
  • Галерея: <section class="hero-gallery"> з зображеннями
  • Товари: <section class="products"> з сіткою карток товарів
  • Промо-банер: <section class="promo-banner">
  • Контакти: <section class="contact-section"> з формою

2.3. Нижній колонтитул (footer)

Реалізовано семантичний тег <footer> з:

  • Структурованою інформацією у вигляді grid-сітки
  • Розділами: "Наш магазин", "Компанія", "Адреса"
  • Контактними посиланнями та адресою

3. Функціонально-стилістичні вимоги

3.1. Адаптивна навігація з бургер-меню

Реалізація:

HTML-код бургер-меню:

<button class="burger-menu" aria-label="Меню">
    <span></span>
    <span></span>
    <span></span>
</button>

CSS-стилі:

.burger-menu {
    display: none;
    flex-direction: column;
    background: transparent;
    border: none;
    cursor: pointer;
    padding: 5px;
    z-index: 1001;
}

JavaScript-функціонал: (файл script.js)

document.addEventListener('DOMContentLoaded', function() {
    const burgerMenu = document.querySelector('.burger-menu');
    const navLinks = document.querySelector('.nav-links');
    
    burgerMenu.addEventListener('click', function() {
        navLinks.classList.toggle('active');
        burgerMenu.classList.toggle('active');
    });
});

Особливості:

  • Бургер-меню з'являється на екранах до 1200px
  • Плавна анімація перетворення іконки при кліку
  • Меню виїжджає зліва з плавним переходом
  • Автоматичне закриття меню при кліку на посилання

3.2. Адаптивна сітка контенту (flexbox/grid)

Використання CSS Grid:

.product-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
    gap: 30px;
}

Використання Flexbox:

nav {
    display: flex;
    justify-content: space-between;
    align-items: center;
}

Переваги:

  • Динамічна адаптація кількості колонок залежно від ширини екрана
  • Автоматичне вирівнювання елементів
  • Гнучкість та масштабованість

3.3. Медіа-запити (media queries)

Реалізовані breakpoints:

  • @media (max-width: 1200px): Адаптація навігації, поява бургер-меню
  • @media (max-width: 1024px): Адаптація для планшетів
  • @media (max-width: 768px): Адаптація для мобільних пристроїв
  • @media (orientation: landscape): Адаптація для ландшафтної орієнтації

Приклад медіа-запиту:

@media (max-width: 768px) {
    .burger-menu {
        display: flex;
    }
    
    .nav-links {
        position: fixed;
        left: -100%;
        top: 70px;
        flex-direction: column;
        width: 100%;
        transition: 0.3s;
    }
    
    .nav-links.active {
        left: 0;
    }
}

3.4. Відносні одиниці вимірювання

Використані відносні одиниці:

  • % - для ширини контейнерів та елементів (width: 100%)
  • rem - для розмірів шрифтів та відступів (font-size: 2rem, padding: 2rem)
  • em - для деяких розмірів відносно батьківського елемента
  • vw/vh - можуть бути використані для адаптації до viewport

Приклади:

.container {
    max-width: 1200px;
    padding: 2rem 40px;
}

section h2 {
    font-size: 2rem;
    margin-bottom: 1rem;
}

3.5. Візуальні ефекти та анімації

Реалізовані ефекти:

  • Плавні переходи: transition: transform 0.3s, box-shadow 0.3s
  • Ефект підняття карток: transform: translateY(-5px) при hover
  • Зміна тіні: box-shadow: 0 8px 16px rgba(0,0,0,0.15)
  • Анімація зображень: transform: scale(1.05) translateY(-10px)
  • Плавна зміна кольорів: transition: color 0.3s
  • Анімація бургер-меню: плавне перетворення іконки при кліку

Приклад анімації:

.product-card {
    transition: transform 0.3s, box-shadow 0.3s;
}

.product-card:hover {
    transform: translateY(-5px);
    box-shadow: 0 8px 16px rgba(0,0,0,0.15);
}

4. Файлова структура веб-застосунку

Організація файлів:

  • HTML-документи: окремо в кореневій директорії
    • index.html - головна сторінка
    • description.html - опис застосунку
    • reports.html - сторінка звітів
  • Стилі: окремий файл styles.css в кореневій директорії
  • Скрипти: окремий файл script.js для функціоналу бургер-меню
  • Графічні ресурси: окрема папка images/ з усіма зображеннями
  • Звіти: окрема папка reports/ з HTML-звітами

Висновок: Файлова структура логічно організована та відповідає вимогам. Всі ресурси зберігаються окремо відповідно до їх призначення.

5. Код бургер-меню

5.1. HTML-код

<button class="burger-menu" aria-label="Меню">
    <span></span>
    <span></span>
    <span></span>
</button>

5.2. CSS-код

.burger-menu {
    display: none;
    flex-direction: column;
    background: transparent;
    border: none;
    cursor: pointer;
    padding: 5px;
    z-index: 1001;
}

.burger-menu span {
    width: 25px;
    height: 3px;
    background-color: #333;
    margin: 3px 0;
    transition: 0.3s;
    border-radius: 3px;
}

.burger-menu.active span:nth-child(1) {
    transform: rotate(-45deg) translate(-5px, 6px);
}

.burger-menu.active span:nth-child(2) {
    opacity: 0;
}

.burger-menu.active span:nth-child(3) {
    transform: rotate(45deg) translate(-5px, -6px);
}

5.3. JavaScript-код

document.addEventListener('DOMContentLoaded', function() {
    const burgerMenu = document.querySelector('.burger-menu');
    const navLinks = document.querySelector('.nav-links');
    
    if (burgerMenu && navLinks) {
        burgerMenu.addEventListener('click', function() {
            navLinks.classList.toggle('active');
            burgerMenu.classList.toggle('active');
        });
        
        const links = navLinks.querySelectorAll('a');
        links.forEach(link => {
            link.addEventListener('click', function() {
                navLinks.classList.remove('active');
                burgerMenu.classList.remove('active');
            });
        });
    }
});

7. Висновки

Під час виконання лабораторної роботи №3 було реалізовано верстку власного сайту з повною відповідністю всім вимогам щодо адаптивності та функціональності.

Основні досягнення:

  1. Створено семантичну структуру HTML з використанням тегів <header>, <nav>, <section>, <footer>
  2. Реалізовано адаптивну навігацію з бургер-меню, яка трансформується на пристроях з малою роздільною здатністю
  3. Застосовано CSS Grid та Flexbox для створення адаптивної сітки контенту
  4. Додано медіа-запити для адаптації стилів під різні розміри екранів (десктоп, планшет, смартфон)
  5. Використано відносні одиниці вимірювання (% , rem, em) для забезпечення гнучкості інтерфейсу
  6. Реалізовано візуальні ефекти та анімації (плавне відкриття меню, зміна стану елементів при наведенні)
  7. Організовано логічну файлову структуру з окремими файлами для HTML, CSS, JavaScript та ресурсів

Отримані знання та навички дозволяють створювати сучасні адаптивні веб-сайти, які коректно відображаються на різних пристроях та розмірах екранів. Розуміння принципів адаптивної верстки, використання медіа-запитів та реалізація інтерактивних елементів є важливими для створення доступних та зручних веб-додатків.
