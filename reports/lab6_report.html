2. ЗАВДАННЯ №2

Використовуючи сценарій на JavaScript, виконати завдання: створити HTML-розмітку, яка складається з наступних елементів: текст, кнопка, інпут (поле введення). Кнопка "Приховати" ховає текст, виводячи зірочки замість введеної інформації, замінює назву кнопки на "Розкрити", при повторному натисканні текст знову стає доступним і кнопка набуває початкового вигляду.

2.1. Програмний код

// Завдання 2: Показ/приховування пароля
// Кнопка "Приховати" ховає текст, показуючи зірочки,
// замінює назву кнопки на "Розкрити",
// при повторному натисканні текст знову стає доступним

document.addEventListener('DOMContentLoaded', function() {
    // Отримуємо посилання на елементи
    const passwordInput = document.getElementById('passwordInput');
    const toggleButton = document.getElementById('toggleButton');

    // Функція для оновлення стану кнопки та поля введення
    function updateToggleState() {
        if (passwordInput.type === 'password') {
            // Текст прихований (зірочки) - кнопка повинна показувати "Розкрити"
            toggleButton.textContent = 'Розкрити';
        } else {
            // Текст видимий - кнопка повинна показувати "Приховати"
            toggleButton.textContent = 'Приховати';
        }
    }

    // Встановлюємо початковий стан: текст видимий, кнопка "Приховати"
    passwordInput.type = 'text';
    updateToggleState();

    // Обробник події кліку на кнопку
    toggleButton.addEventListener('click', function() {
        // Перемикаємо тип поля введення між "password" та "text"
        if (passwordInput.type === 'password') {
            // Якщо текст прихований - робимо його видимим
            passwordInput.type = 'text';
        } else {
            // Якщо текст видимий - ховаємо його (показуємо зірочки)
            passwordInput.type = 'password';
        }

        // Оновлюємо текст кнопки
        updateToggleState();
    });
});

2.2. Пояснення коду

Основні елементи коду:

  • document.addEventListener('DOMContentLoaded', ...) - обробник події, який чекає повного завантаження HTML-документу перед виконанням коду. Це гарантує, що всі елементи вже існують в DOM
  • document.getElementById('passwordInput') - отримання посилання на елемент input за його ID. Використовується для доступу до поля введення пароля
  • document.getElementById('toggleButton') - отримання посилання на кнопку перемикання
  • passwordInput.type - властивість, яка визначає тип поля введення. Може бути 'text' (текст видимий) або 'password' (текст прихований зірочками)
  • toggleButton.textContent - властивість для отримання або встановлення текстового вмісту кнопки
  • addEventListener('click', ...) - додавання обробника події кліку на кнопку. При натисканні виконується функція
  • if-else - умовний оператор для перевірки поточного стану поля (приховане чи видиме) та зміни його типу
  • updateToggleState() - функція для синхронізації тексту кнопки з поточним станом поля введення

2.4. Логіка роботи

  1. Початковий стан: Поле має тип 'text', тому текст видимий. Кнопка відображає текст "Приховати"
  2. При натисканні "Приховати": Тип поля змінюється на 'password', текст стає прихованим (відображаються зірочки). Текст кнопки змінюється на "Розкрити"
  3. При натисканні "Розкрити": Тип поля змінюється на 'text', текст стає видимим. Текст кнопки змінюється на "Приховати"
  4. Цикл повторюється: Кожне натискання кнопки перемикає стан між видимим та прихованим текстом

4.4. Пояснення ключових концепцій

Делегування подій (Event Delegation):

  • Концепція: Замість додавання обробника подій до кожного елемента окремо, ми додаємо один обробник до батьківського елемента (контейнера). Події від дочірніх елементів "спливають" (bubble up) до батьківського елемента.
  • Переваги:
    • Менше обробників подій в пам'яті
    • Працює з динамічно доданими елементами
    • Кращі показники продуктивності
    • Спрощений код
  • Реалізація: Використовуємо event.target.closest('.product-card') для визначення, на яку картку товару було клікнуто.

Модальне вікно (Modal Window):

  • Призначення: Вспливаюче вікно, яке відображається поверх основного контенту для показу додаткової інформації або взаємодії з користувачем.
  • Реалізація:
    • Використання position: fixed для позиціонування вікна
    • Напівпрозорий фон (backdrop) для акценту на модальному вікні
    • Блокування прокрутки основного контенту через document.body.style.overflow = 'hidden'
    • Анімації для плавного появи/зникнення

Методи та властивості, що використовуються:

  • querySelector() - вибір першого елемента за селектором
  • getElementById() - отримання елемента за ID
  • addEventListener() - додавання обробника подій
  • event.target - елемент, на який було клікнуто
  • closest() - пошук найближчого батьківського елемента за селектором
  • classList.add/remove - додавання/видалення CSS-класів
  • querySelector() всередині елемента - пошук дочірніх елементів
  • createElement() - створення нового DOM-елемента
  • appendChild() - додавання дочірнього елемента
  • innerHTML - встановлення/отримання HTML-вмісту
  • textContent - встановлення/отримання текстового вмісту
  • getAttribute() - отримання значення атрибута
  • split() - розділення рядка на масив
  • pop() - отримання останнього елемента масиву

5. ДЕТАЛЬНИЙ РОЗБІР КОДУ ЗАВДАННЯ №1

5.1. Функція openModal()

function openModal(productCard) {
    // Отримуємо посилання на елементи картки товару
    const productImage = productCard.querySelector('.product-image');
    const productTitle = productCard.querySelector('.product-title');
    const productPrice = productCard.querySelector('.product-price');
    const productDescription = productCard.querySelector('.product-description');
    const colorOptions = productCard.querySelectorAll('.color-dot');

    // Отримуємо шлях до зображення та визначаємо ключ товару
    const imgSrc = productImage.getAttribute('src');
    const productKey = getProductKey(imgSrc);
    const productData = productsData[productKey];

    // Заповнюємо модальне вікно даними
    document.getElementById('modalImage').src = imgSrc;
    document.getElementById('modalImage').alt = productTitle.textContent;

    if (productData) {
        // Використовуємо розширені дані з об'єкта productsData
        document.getElementById('modalTitle').textContent = productData.title;
        document.getElementById('modalPrice').textContent = productData.price;
        document.getElementById('modalDescription').textContent = productData.description;

        // Створюємо елементи для кольорів
        const modalColors = document.getElementById('modalColors');
        modalColors.innerHTML = '';
        productData.colors.forEach(color => {
            const colorDot = document.createElement('div');
            colorDot.className = 'color-dot';
            colorDot.style.backgroundColor = color;
            if (color === '#FFFFFF') {
                colorDot.style.border = '2px solid #ddd';
            }
            modalColors.appendChild(colorDot);
        });
    } else {
        // Використовуємо дані з HTML, якщо немає розширених даних
        document.getElementById('modalTitle').textContent = productTitle.textContent;
        document.getElementById('modalPrice').textContent = productPrice.textContent;
        document.getElementById('modalDescription').textContent = productDescription.textContent;

        const modalColors = document.getElementById('modalColors');
        modalColors.innerHTML = '';
        colorOptions.forEach(colorDot => {
            const newColorDot = colorDot.cloneNode(true);
            newColorDot.style.width = '32px';
            newColorDot.style.height = '32px';
            modalColors.appendChild(newColorDot);
        });
    }

    // Показуємо модальне вікно та блокуємо прокрутку
    modal.classList.add('active');
    document.body.style.overflow = 'hidden';
}

5.2. Реалізація делегування подій

// Делегування подій на product-grid
if (productGrid) {
    productGrid.addEventListener('click', function(event) {
        // event.target - елемент, на який було клікнуто
        // closest() шукає найближчий батьківський елемент з класом .product-card
        const productCard = event.target.closest('.product-card');

        if (productCard) {
            // Перевіряємо, що клік не був на кнопці "Додати до кошика"
            if (!event.target.classList.contains('btn-add-cart')) {
                openModal(productCard);
            }
        }
    });
}

Як це працює:

  1. Обробник подій додається тільки до одного елемента - productGrid
  2. Коли користувач клікає на будь-який елемент всередині productGrid, подія "спливає" до батьківського елемента
  3. За допомогою event.target.closest('.product-card') визначаємо, на яку картку товару було клікнуто
  4. Якщо знайдено картку товару і клік не був на кнопці "Додати до кошика", відкриваємо модальне вікно

5.3. Функція закриття модального вікна

function closeModal() {
    modal.classList.remove('active');
    document.body.style.overflow = ''; // Відновити прокрутку
}

// Закриття при кліку на хрестик
modalClose.addEventListener('click', closeModal);

// Закриття при кліку поза модальним вікном
modal.addEventListener('click', function(event) {
    if (event.target === modal) {
        closeModal();
    }
});

// Закриття при натисканні Escape
document.addEventListener('keydown', function(event) {
    if (event.key === 'Escape' && modal.classList.contains('active')) {
        closeModal();
    }
});

6. ПЕРЕВАГИ ВИКОРИСТАННЯ ДЕЛЕГУВАННЯ ПОДІЙ

1. Ефективність пам'яті:

Замість N обробників подій (де N - кількість товарів), ми маємо лише один обробник. Це особливо важливо при великій кількості елементів.

2. Робота з динамічним контентом:

Якщо товари додаються на сторінку динамічно (через AJAX, через JavaScript), делегування подій продовжує працювати без додаткових налаштувань.

3. Спрощений код:

Не потрібно додавати обробники до кожного елемента окремо в циклі. Код стає чистішим та легшим для підтримки.

4. Краща продуктивність:

Менша кількість обробників подій означає менше витрат ресурсів браузера на їх управління.

5. Гнучкість:

Легко додавати або видаляти елементи з колекції - обробник подій автоматично працює з новими елементами.

7. АДАПТИВНІСТЬ ТА UX

7.1. Адаптивний дизайн

Модальне вікно адаптується до різних розмірів екранів:

  • Планшети та десктопи (>= 769px): Горизонтальне розташування зображення та тексту

7.2. Покращення користувацького досвіду

  • Плавні анімації появи/зникнення модального вікна
  • Блокування прокрутки основного контенту при відкритому модальному вікні
  • Кілька способів закриття (хрестик, клік поза вікном, клавіша Escape)
  • Візуальний фокус на модальному вікні завдяки напівпрозорому фону
  • Можливість прокрутки вмісту модального вікна при великому обсязі інформації
